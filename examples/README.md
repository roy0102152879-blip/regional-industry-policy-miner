# Examples

These examples are simplified public case cards. They are not full due-diligence
reports. Their purpose is to show what a good run of `regional-industry-policy-miner`
should decide, what evidence layers it should preserve, and what follow-up work
should happen next.

## Case Index

| case | mode | why it matters | public card |
| --- | --- | --- | --- |
| Jiaxing regional scan | Mode B: regional blind scan | Shows a mature advanced-manufacturing city where photovoltaic/new energy, new materials, and integrated circuits can be ranked by policy signal plus enterprise carrying. | [`jiaxing_simplified.md`](jiaxing_simplified.md) |
| Shaya County regional scan | Mode B: regional blind scan | Shows a county/resource-region pattern where cotton full-chain, oil and gas chemical processing, and PV sand-control must be separated instead of using generic "advantaged industry" language. | [`shaya_county_simplified.md`](shaya_county_simplified.md) |

## What These Cases Validate

- A regional scan should answer what the local government is really betting on.
- Candidate industries should be ranked, not just listed.
- Policy support, local carrying capacity, and commercial-quality pre-check should stay separate.
- Representative enterprises or project subjects should be included for top industries.
- A strong Mode B result should point to the next Mode A deep dive.

## Full-Run Discipline

For formal use, each case should be expanded into:

- a source library with `doc_id`, title, date, source type, and URL;
- a candidate scoring table;
- an enterprise/project mapping table;
- a quantitative layer that separates actuals from targets;
- risks, counterexamples, and data gaps.
